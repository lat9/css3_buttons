What it does

This plugin replaces Zen-Cart's default CSS button functionality (prior to v1.5.3) with some CSS3 styling, based on the initial output provided by http://www.cssbuttongenerator.com/. The buttons produced have rounded corners in the more recent browsers and "fall back" to squared corners on non-CSS3 aware browsers. If your store is running Zen Cart v1.5.3 or later, you should install this plugin only if you plan to use its integration with Font Awesome glyphs.

How to use it

First, install the plugin using the following steps:

  1. Rename the plugin's YOUR_ADMIN folder to match your store's admin folder name.
  2. Rename the plugin's YOUR_TEMPLATE folder to match your store's current template name.
  3. If the file /includes/functions/html_output.php is modified from your Zen Cart version, use a file-merge program (like WinMerge or Beyond Compare) to merge your store's special changes into the plugin's version of the file. The plugin uses the file version from Zen Cart v1.5.3, with three (3) marked change sections to the zenCssButton function.
  4. If the file /includes/templates/YOUR_TEMPLATE/common/html_header.php exists, use a file-merge program to merge your store's special changes into the plugin's version of the file. The plugin uses the file version from Zen Cart v1.5.3, with one (1) marked change section.
  5. Copy the plugin's files to your store's file system.
  6. Log into your Zen Cart admin. This will cause the initialization script for the CSS buttons to run, adding the items identified below to your admin's Configuration->Layout Settings.

Configuration->Layout Settings Additions

Once you've copied the plugin's files and signed into your Zen Cart admin, the following configuration settings are added to your admin's Configuration->Layout Settings. Make sure that the CSS Buttons setting's value is True to enable the CSS3 buttons.

Title Description Default
Include a link to the Font Awesome stylesheet? If true, a link to the Font Awesome CSS file will be included in each page's header. If your template already includes Font Awesome, leave this setting false. false
Font Awesome version number Enter the Font Awesome version number to load, if the previous setting is set to true. 4.2.0
Submit-button HTML Tag Identify the type of HTML tag to use for CSS-generated submit buttons, either input (default) or button. Please note that the button tag is not fully supported on older browsers. input

To include Font Awesome glyphs in your buttons, edit the file /includes/extra_definitions/css_button_glyphs.php to assign the glyphs on a button-by-button basis using the array defined within that script. The "Send" button below shows the formatting displayed when Submit-button HTML Tag is set to button. In all cases, the glyph you define is prepended to the button's name.

Note: Font Awesome glyphs are not applied when Submit-button HTML Tag is set to input.

The button text for the built-in Zen-Cart buttons is found in the alt-tag text in /includes/languages/english/YOUR_TEMPLATE/button_names.php. For example, the text for the "Send" button above comes from this PHP define:

define('BUTTON_SEND_ALT', 'Send');

Note: Zen Cart's standard processing in the function zen_image_submit (also in /includes/functions/html_output.php) forces a default to the button's image if the button's alt-text is more than 30 characters in length. This "trick" only applies to the submit-type buttons; a similar check does not occur for link-type buttons!

Note 2: You can style an individual button by using the button's name as a selector. Remember to place these definitions at the bottom of the stylesheet_css_buttons.css file and make sure that the non-Hover styling is specified before the Hover styling: 

span.normal_button.button_sold_out_sm {
  background-color: blue;
}
span.normal_button.button_sold_out_smHover {
  background-color: red;
}
